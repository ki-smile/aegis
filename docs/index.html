Three-module architecture

DARM manages data, MMM monitors performance and drift, CDM makes structured decisions.

DARM

Dataset Assimilation and Retraining Module. Manages training (DT,k), golden (DG), and drifting (DD,k) datasets with unconditional accumulation.

MMM

Model Monitoring Metrics. Computes MLcps composite score, drift detection via Bonferroni-corrected KS test, and fairness metrics.

CDM

Clinical Decision Module. Priority-ordered decision hierarchy (P1–P4) with independent PMS alarm channel (A1–A3).

Decision taxonomy

Four deployment decision categories with composite output — APPROVE + ALARM is not contradictory.

APPROVE CONDITIONAL APPROVAL CLINICAL REVIEW ALARM REJECT
Priority Condition Decision Regulatory action
P1 (highest) Any metric < Pfail REJECT Suspend model; initiate CAPA
P2 Performance buffer / regression from Pref CLINICAL REVIEW Hold deployment; clinician review required
P3 Minor drift or TAI concern CONDITIONAL APPROVAL Deploy with enhanced monitoring
P4 (lowest) All checks pass APPROVE Release model; routine PMS
